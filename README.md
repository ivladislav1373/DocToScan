**DocToScan** — утилита для преобразования документов (PDF, DOCX) в стилизованные "скан-копии" с эффектами старого сканера: регулировка яркости, случайный поворот страниц, чёрно-белый режим и обрезка до заданного формата (A4, A3, Letter, Legal).

## Возможности

- ✅ Поддержка PDF и DOCX (через Microsoft Word)
- ✅ Пакетная обработка нескольких файлов
- ✅ Настраиваемые эффекты: яркость, поворот, градации серого
- ✅ Принудительный размер страницы (A4, A3, Letter, Legal или произвольный)
- ✅ Интеграция в контекстное меню Проводника
- ✅ Автономная сборка (self-contained), не требует установленного .NET Runtime
- ✅ Полная поддержка кириллицы в путях и интерфейсе

## Требования

- **Операционная система**: Windows 10/11 (64-bit)
- **Для работы с DOCX**: Microsoft Word 2010 или новее (2013, 2016, 2019, 2021, Microsoft 365)
- **Для сборки из исходников**: .NET 10.0 SDK

## Установка

### Вариант 1: Готовый дистрибутив

1. Скачайте последний архив из раздела [Releases](https://github.com/ivladislav1373/DocToScan/releases/) (или соберите самостоятельно).
2. Распакуйте архив в любую папку (рекомендуется `C:\Program Files\DocToScan`).
3. Запустите `install.bat` **от имени администратора** для интеграции в контекстное меню.

### Вариант 2: Сборка из исходников

```powershell
# Клонируйте репозиторий
git clone https://github.com/ivladislav1373/DocToScan.git
cd DocToScan

# Запустите скрипт сборки
.\scripts\build-release.ps1
```

После сборки архив появится в папке `builds/`.

## Использование

### Через контекстное меню

1. Нажмите правой кнопкой мыши на один или несколько PDF/DOCX файлов.
2. Выберите пункт **«Создать скан-копию»**.
3. Программа создаст в той же папке файлы с суффиксом `(скан).pdf`.

### Из командной строки

```cmd
DocToScan.exe "C:\Docs\report.docx" "C:\Docs\scan.pdf"
```

Можно передавать несколько файлов одновременно.

### Специальные команды

- `DocToScan.exe /install` — установка контекстного меню (требует прав администратора)
- `DocToScan.exe /uninstall` — удаление контекстного меню

## Конфигурация

Все настройки хранятся в файле `config.xml` в папке программы. Пример:

```xml
<Configuration>
  <Brightness>
    <Enable>true</Enable>
    <Level>15</Level>           <!-- от -255 до 255 -->
  </Brightness>
  
  <Rotation>
    <Enable>true</Enable>
    <MinAngle>-3</MinAngle>     <!-- градусы, отрицательный = влево -->
    <MaxAngle>3</MaxAngle>      <!-- положительный = вправо -->
  </Rotation>
  
  <Grayscale>
    <Enable>true</Enable>
  </Grayscale>
  
  <ImageQuality>
    <Dpi>150</Dpi>              <!-- 72–600 -->
    <JpegCompression>85</JpegCompression>   <!-- 1–100 -->
  </ImageQuality>
  
  <PageSize>
    <Enable>true</Enable>
    <WidthMm>210</WidthMm>      <!-- A4 ширина -->
    <HeightMm>297</HeightMm>    <!-- A4 высота -->
  </PageSize>
</Configuration>
```

При первом запуске файл создаётся автоматически с настройками по умолчанию.

## Скрипты

В папке `scripts/` находятся служебные файлы:

| Скрипт | Назначение |
|--------|------------|
| `build-release.ps1` | Сборка дистрибутива и создание ZIP-архива |
| `install.bat` | Установка интеграции в контекстное меню (с само-повышением прав) |
| `uninstall.bat` | Удаление интеграции (с само-повышением прав) |
| `check-word.ps1` | Проверка наличия и доступности Microsoft Word |

Подробнее о каждом скрипте — в [scripts/README.md](scripts/README.md).

## Сборка дистрибутива

Для сборки требуется .NET 10.0 SDK. Запустите:

```powershell
.\scripts\build-release.ps1 [-Configuration Release] [-Version 1.2.3] [-OutputDir builds]
```

Параметры:
- `Configuration` — `Release` или `Debug` (по умолчанию Release)
- `Version` — версия сборки (если не указана, берётся из `Directory.Build.props`)
- `OutputDir` — папка для результатов (по умолчанию `builds`)

Результат: ZIP-архив вида `DocToScan_vВерсия_Дата.zip` в папке `builds/`.

## Устранение неполадок

### «Не удаётся найти pdfium.dll»
- Убедитесь, что в папке с программой есть файл `pdfium.dll` или папка `x64/pdfium.dll`.
- При сборке скрипт автоматически копирует библиотеку из пакета NuGet.

### «Microsoft Word не найден» при работе с DOCX
- Установите Microsoft Word (любая версия от 2010).
- Запустите `check-word.ps1` для диагностики.
- Для Office 365 убедитесь, что установлены компоненты для разработки (Primary Interop Assemblies).

### Контекстное меню не появляется
- Запустите `install.bat` от имени администратора.
- Проверьте, не блокирует ли антивирус изменения реестра.

## Заметки для разработчиков

### Потенциальные улучшения / известные недочёты

1. **Загрузка Word через COM**  
   При отсутствии Word или его повреждении программа выдаёт ошибку. Добавлен детальный лог и код возврата в `check-word.ps1`, но можно улучшить диагностику в рантайме.

2. **Обработка очень больших PDF**  
   Рендеринг всех страниц сразу может потреблять много памяти. В будущем можно реализовать постраничную обработку с записью на диск.

3. **Поворот страниц**  
   Случайный угол применяется один раз для всех страниц (как в старом сканере). Если нужно разное вращение на каждой странице, можно добавить опцию.

4. **Single‑file предупреждение**  
   При сборке с `PublishSingleFile=true` возникает предупреждение IL3000 в `Program.cs`. Используется `AppContext.BaseDirectory` как fallback, но для полной совместимости стоит заменить все обращения к `Assembly.Location`.

5. **Очистка временных файлов**  
   При аварийном завершении некоторые временные файлы могут остаться. `TempFileManager` регистрирует созданные файлы, но если процесс убит принудительно, очистка не происходит.

6. **Тестирование**  
   В проекте пока нет модульных тестов. Рекомендуется добавить тесты для конвертации, эффектов и генерации PDF.

Эти замечания не критичны для текущей версии, но могут быть учтены в будущих релизах.
