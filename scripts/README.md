# Скрипты DocToScan

## 📋 Описание

Этот каталог содержит вспомогательные скрипты для сборки, установки и диагностики программы DocToScan.

## 🚀 Доступные скрипты

### Для пользователей

| Скрипт | Назначение | Требует прав |
|--------|------------|--------------|
| `install.bat` | Установка интеграции в контекстное меню | Администратор |
| `uninstall.bat` | Удаление интеграции из контекстного меню | Администратор |
| `check-office.ps1` | Диагностика установленной версии Microsoft Office | Нет |

### Для разработчиков

| Скрипт | Назначение |
|--------|------------|
| `build-release.ps1` | Сборка релизной версии и создание дистрибутива |
| `fix-encoding.ps1` | Исправление кодировки всех скриптов (UTF-8) |

## 🔧 Использование

### Установка программы
```batch
# Запустите от имени администратора
install.bat
```

### Удаление программы
```batch
# Запустите от имени администратора
uninstall.bat
```

### Проверка совместимости с Office
```powershell
# Запустите в PowerShell
.\check-office.ps1
```

### Сборка дистрибутива
```powershell
# Запустите в PowerShell из корня проекта
.\scripts\build-release.ps1
```

## ⚙️ Особенности

- Все скрипты поддерживают **кириллицу** (UTF-8)
- `.bat` файлы используют кодировку UTF-8 с BOM
- `.ps1` файлы используют UTF-8 без BOM

## 🐛 Решение проблем

### Если в консоли отображаются "квадратики" вместо букв:

1. Убедитесь, что консоль поддерживает UTF-8:
   ```batch
   chcp 65001
   ```

2. Запустите скрипт `fix-encoding.ps1`:
   ```powershell
   .\scripts\fix-encoding.ps1
   ```

3. В Windows Terminal проверьте настройки профиля:
   - Параметр "Кодировка" должен быть UTF-8

### Если скрипты не запускаются в PowerShell:

```powershell
# Разрешить выполнение скриптов
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 📝 Примечания

- Для установки/удаления требуются права администратора
- Диагностика Office не требует прав администратора
- Все скрипты протестированы на Windows 10/11 (64-bit)
