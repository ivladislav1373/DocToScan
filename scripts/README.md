# DocToScan - Скрипты

Этот каталог содержит служебные скрипты для работы с программой DocToScan.

## 📦 Сборка дистрибутива

### `build-release.ps1`
Скрипт для сборки проекта и создания ZIP-архива с дистрибутивом.

**Использование:**
```powershell
.\build-release.ps1 [-Configuration Release|Debug] [-Version X.X.X] [-OutputDir builds]
```

**Параметры:**
- `Configuration` - конфигурация сборки (по умолчанию Release)
- `Version` - версия сборки (если не указана, берётся из Directory.Build.props)
- `OutputDir` - директория для выходных файлов (по умолчанию builds)

**Примеры:**
```powershell
.\build-release.ps1
.\build-release.ps1 -Version 1.2.3
.\build-release.ps1 -Configuration Debug -OutputDir mybuilds
```

**Результат:** Создаёт архив `DocToScan_vВерсия_Дата.zip` в папке `builds/`

## 🔧 Установка и удаление

### `install.bat`
Устанавливает интеграцию DocToScan в контекстное меню Windows.

**Использование:**
- Двойной клик мышью
- Или запуск из командной строки: `install.bat`

**Что делает:**
- ✅ Автоматически запрашивает права администратора
- ✅ Добавляет пункт "Создать скан-копию" в меню PDF и DOCX файлов
- ✅ Создаёт папки Logs и Temp
- ✅ Опционально добавляет программу в PATH

### `uninstall.bat`
Удаляет интеграцию DocToScan из контекстного меню.

**Использование:**
- Двойной клик мышью
- Или запуск из командной строки: `uninstall.bat`

**Что делает:**
- ✅ Автоматически запрашивает права администратора
- ✅ Удаляет пункты из контекстного меню
- ✅ Очищает записи в реестре
- ✅ Удаляет программу из PATH
- ✅ Опционально удаляет файлы программы и данные

## 🔍 Проверка системы

### `check-word.ps1`
Проверяет наличие и доступность Microsoft Word для работы с DOCX файлами.

**Использование:**
```powershell
.\check-word.ps1 [-Quiet] [-Version]
```

**Параметры:**
- `Quiet` - тихий режим (только код возврата)
- `Version` - показать только версию Word

**Коды возврата:**
- `0` - Word найден и совместим
- `1` - Word найден, но COM недоступен
- `2` - Word найден, но версия старая (< 12.0)
- `3` - Word не найден

**Примеры:**
```powershell
.\check-word.ps1
.\check-word.ps1 -Version
if (.\check-word.ps1 -Quiet) { echo "Word готов" }
```

## 📋 Примечания

- Все скрипты поддерживают кириллицу (UTF-8)
- Для работы `install.bat` и `uninstall.bat` требуются права администратора
- Скрипт `check-word.ps1` не требует прав администратора
- `build-release.ps1` требует .NET SDK 10.0

## 🚨 Возможные проблемы

**1. Скрипты не запускаются**
- Разрешите выполнение скриптов PowerShell: `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

**2. Ошибки кодировки**
- Убедитесь, что файлы сохранены в UTF-8 with BOM
- Консоль должна поддерживать Unicode (шрифт Consolas)

**3. Word не определяется**
- Убедитесь, что Microsoft Word установлен
- Для Office 365 может потребоваться дополнительная настройка COM
- Проверьте разрядность Word (должна совпадать с разрядностью Windows)
